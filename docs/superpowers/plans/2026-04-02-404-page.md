# 404 Page Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Create a retro pixel-art 404 page where a CSS T-Rex enters from the right, destroys the "404" heading, and a humorous message fades in — all with zero JavaScript and no external images.

**Architecture:** Single `src/pages/404.astro` file using the existing `Layout.astro` wrapper. The T-Rex is drawn with CSS `box-shadow` pixel art on `::before`/`::after` pseudo-elements. All animations are pure `@keyframes` CSS with carefully timed `animation-delay` values to sequence the scene.

**Tech Stack:** Astro, inline `<style>` CSS, `@keyframes` animations, CSS `box-shadow` pixel art, Tailwind (for layout classes only)

### Task 3: Draw the CSS pixel-art T-Rex

**Files:**
- Modify: `src/pages/404.astro` — `<style>` block

The T-Rex is 10 pixels wide × 8 pixels tall, each pixel rendered as a `box-shadow` on `::before` (body) and `::after` (legs). Scale of 10 means each "pixel" = 10px → dino is 100px × 80px. The `dino-wrap` sizes the container to accommodate the scaled output.

Pixel grid (facing LEFT, head on left side):
```
Row 1:  cols 1-4          (head top)
Row 2:  cols 1-6          (head)
Row 3:  cols 1-6          (head, col 1 = white eye)
Row 4:  cols 1-8          (jaw)
Row 5:  cols 1-10         (body)
Row 6:  cols 1-4, 7-8     (lower body + tail)
Row 7:  legs (animated)
Row 8:  feet (animated)
```

- [ ] **Step 1: Add `.dino-wrap` and `.dino` styles inside `<style>` (after `.btn-home:hover`)**

```css
  /* ─── T-Rex Pixel Art ──────────────────────────────── */
  .dino-wrap {
    position: relative;
    z-index: 1;
    width: 120px;   /* 10px scale × 12 cols max */
    height: 90px;   /* 10px scale × 9 rows max  */
    margin-bottom: 0;
  }

  .dino {
    position: absolute;
    top: 0;
    left: 0;
  }

  /* Body pixels (static) on ::before */
  .dino::before {
    content: '';
    position: absolute;
    width: 10px;
    height: 10px;
    box-shadow:
      /* Row 1 — head top */
```

### Task 5: Entry animation + 404 destruction sequence

**Files:**
- Modify: `src/pages/404.astro` — `<style>` block

This task wires up the full scene sequence:
1. Dino slides in from the right (0.5s start)
2. 404 shakes (0.8s delay)
3. Each digit explodes outward (1.1s delay, different directions)
4. Dino bobs (idle, 1.3s delay)
5. Message fades up (1.5s delay)

- [ ] **Step 1: Add all sequence `@keyframes` to `<style>`**

```css
  /* ─── Entry: dino slides in from right ────────────── */
  @keyframes dino-enter {
    from { transform: translateX(110vw); }
    to   { transform: translateX(0);     }
  }

  /* ─── Idle: subtle vertical bob ───────────────────── */
  @keyframes dino-idle {
    0%, 100% { transform: translateX(0) translateY(0);   }
    50%       { transform: translateX(0) translateY(-6px); }
  }

  /* ─── 404 shake ────────────────────────────────────── */
  @keyframes shake {
    0%,100% { transform: translateX(0);    }
    10%     { transform: translateX(-8px); }
    20%     { transform: translateX(8px);  }
    30%     { transform: translateX(-6px); }
    40%     { transform: translateX(6px);  }
    50%     { transform: translateX(-4px); }
    60%     { transform: translateX(4px);  }
    70%     { transform: translateX(-2px); }
    80%     { transform: translateX(2px);  }
    90%     { transform: translateX(0);    }
  }

  /* ─── Individual digit explosions ──────────────────── */
  @keyframes explode-1 {
    0%   { transform: translate(0, 0) rotate(0deg);            opacity: 1; }
    100% { transform: translate(-120px, -80px) rotate(-25deg); opacity: 0; }
  }
  @keyframes explode-2 {
    0%   { transform: translate(0, 0) rotate(0deg);       opacity: 1; }
    100% { transform: translate(0, 120px) rotate(15deg);  opacity: 0; }
  }
  @keyframes explode-3 {
    0%   { transform: translate(0, 0) rotate(0deg);              opacity: 1; }
    100% { transform: translate(140px, -60px) rotate(30deg);     opacity: 0; }
  }

  /* ─── Message fade up ───────────────────────────────── */
  @keyframes fade-up {
    from { opacity: 0; transform: translateY(20px); }
    to   { opacity: 1; transform: translateY(0);    }
  }
```

- [ ] **Step 2: Wire animations to elements**

Add the following rules inside `<style>` (after the keyframes):

```css
  /* ─── Animation wiring ─────────────────────────────── */
  .dino-wrap {
    animation:
      dino-enter 0.6s cubic-bezier(0.22, 1, 0.36, 1) 0.4s both,
      dino-idle  1.0s ease-in-out 1.2s infinite;
  }

  .text-404 {
    animation: shake 0.4s ease-in-out 0.9s both;
  }

  .d1 {
    animation: explode-1 0.5s ease-in 1.3s forwards;
  }
  .d2 {
    animation: explode-2 0.5s ease-in 1.35s forwards;
  }
  .d3 {
    animation: explode-3 0.5s ease-in 1.4s forwards;
  }

  .message {
    opacity: 0;
    animation: fade-up 0.6s ease-out 1.8s forwards;
  }
```

- [ ] **Step 3: Verify full animation sequence**

Reload `http://localhost:4321/404`.
Expected sequence:
- Dino slides in from the right ~0.4s after load
- 404 shakes at ~0.9s
- All three digits scatter and fade out at ~1.3s
